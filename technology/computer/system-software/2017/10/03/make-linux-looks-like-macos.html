Linux变macOS

发表于 2017-10-03 | 分类于 technology / computer / system-software | 评论  | 阅读次数 
| 字数统计 1,928

折腾的原因呢，安装的网易云音乐，有一些歌曲不能播放，很怪，于是Google了一大堆，说要安装gstreamer，于是安装了一大堆gstreamer相关的包。但是，发现问题还是没有解决啊，于是卸载呗，突然想起百度经验上面的apt-get介绍，有个强力推荐，于是然后apt-get autoremove gstreamer* --purge卸载呗。卸载了好久啊，想起了之前的rm悲剧，然后真的又悲剧了……

题外话

产生了三个悲剧。

  • 导致EasyScreenCast没用

在Tweak Tool那显示Error，提到什么Required Gst，无奈只好Google这个插件，找到Github页面，看到依赖gstreamer，恍然大悟，赶紧apt-get install gstreamer*安装，最后终于解决。

  • 导致文件浏览器（Nautilus）无法自动显示图片略缩图（thumbnails）

至今还没解决，只有安装gThumb，点击一个文件夹的图片，然后会自动生成当前文件夹的thumbnails，再重新加载文件夹，便能显示。

已解决，apt-cache search gstreamer，把相关的包全部安装。

  • 悲剧它自己

其中折腾EasyScreenCast时，在Gnome Exitesions看到很多插件，于是Google Top Gnome Exitesions，看Youtube视频，突然发现它的界面好好看啊！然后折腾Theme咯，突然想把自己一直Globe Dark的Gnome界面折腾成macOS风格，于是……

Theme

进入正题，先把我的版本贴出来：

我的是GTK3，如果你的不是这个版本，不要紧，下面的不一定不能装，自己去页面查看说明。

首先打开Tweak Tool，里面的GTK+、Shell theme都可以在gnome-look网站上找到，可以找到自己喜欢的下载，然后放到/usr/share/themes目录下，或者~/.themes/（没有自己创建，并且要安装插件User Themes，gnome shell插件安装方法见这篇文章）目录下即可。

tip：新加个Theme，要关闭Tweak Tool再打开才能看到新加的，如果切换后没有效果，重启Gnome，Alt+F2，输入r回车，再不行就重启系统。

GTK+ & Shell theme

下面的三个主题既有GTK+又有Shell的主题，另：有Github地址，且会用Github的，尽量去相应目录直接git clone，这样以后更新就可以直接git pull。

  • Gnome OSX-III-1.0

https://www.gnome-look.org/p/1171688/

把Gnome OSX-III-1.0.tar.gz、Gnome-OSX-Shell-themes-1.0.tar.gz、Extra-1.0.tar.gz（里面有壁纸）都下载，然后将Gnome-OSX-Shell-themes-1.0.tar.gz解压的文件夹里面的Gnome-OSX-Light-Shell里面的gnome-shell文件夹复制到Gnome OSX-III-1.0文件夹里。

安装字体，把两个文件复制到/usr/share/fonts/truetype/目录下，然后fc-cache -f -v即可。

  • OSX Arc White

https://www.gnome-look.org/p/1167049/

下载OSX-Arc-White-v1.4.3.tar.gz，当然这是我现在看到的最新的，如果有新版就下新版。

https://github.com/LinxGem33/OSX-Arc-White

下载Description为macOS Sierra Github (latest)的。

图标安装，gnome-look网站无法直接下载，而是要先安装OCS-Store，然后可以就可以直接安装，OCS-Store会安装到~/.local/share/icons/目录下。

为啥要安装OCS-Store，gnome-look网站页面左上方有个叫Opendesktop的东东，应该就是这个搞的鬼，看起来像是个强大的组织。

当然，有Github页面的，可以直接到/usr/share/icons/目录，或者~/.local/share/icons/目录，然后git clone，以后更新简单。

GDM登录界面主题

GDM登录界面只有通过替换目录/usr/share/gnome-shell/下的gnome-shell-theme.gresource文件实现更改（替换前务必先备份原文件），壁纸自定义可以通过这种方法重新生成gnome-shell-theme.gresource实现，其它元素自定义困难。

建议直接Google搜索gnome-shell-theme.gresource，找一个提供这一文件并且合自己口味GTK主题，比如Flat-Plat（下载后，先执行脚本安装主题，然后才会生成这个文件，再去/usr/share/themes/Flat-Plat/gnome-shell/下找这个文件），OSX-Arc-White（直接下载），然后将文件放到上文所提到的目录即可，壁纸可按上文链接中的方法自定义。

博主的配置

类型主题

为啥GTK+用Gnome OSX-III-1.0？附上页面上该作者的吐槽：

This is a gnome-desktop-interpretation of Mac OSX. I’ve tried to implement the feel of OSX on the gnome-applications. In version 3.0 I’ve modernized it in every little detail. There is nothing (not a single item) that is not new. Resulting in a completely rewritten GTK.CSS-file four times bigger than the previous one, while the theme feels more responsive. I’ve also added a dark theme, so Terminal, Photo’s, and Video’s are automatically dark-themed. I’ve spend a great deal of time (3 months) and effort on this theme into fine-tuning it, so I hope you try before you judge !

的确，这厮用了三个月，界面效果确与其它两个不同，自己对比一下其它两个即能感受出。正如作者所言，细节真的很棒，比如渐变！

但是，Gnome OSX-III-1.0自带的Shell theme不咋地，所以我用了OSX Arc White的Shell theme，没话说，就是赞！

Icons我用的Shadow，感觉很喜欢这个风格，虽然不太好辨认，但是很合我口味！

Terminal用的Gogh的Aci，然后自己再在Performance>Profiles>Aci>Edit自定义了一下。

GDM用的OSX Arc White自带的（将OSX-Arc-White/gnome-shell/gnome-shell-theme.gresource复制到/usr/share/gnome-shell/下，替换原来的，替换前记得先备份），与Shell theme保持一致吧。
